# recordTogif 技术栈

本文档总结当前项目已落地并验证可用的技术方案（开发运行、打包、安装后运行均已通过）。

## 1) 语言与工程基础

- 语言：`Rust`
- 包管理与构建：`cargo`
- 项目类型：桌面端二进制应用（`cargo init --bin`）

## 2) GUI 框架

- GUI：`iced`（纯 Rust）
- UI 形态：单窗口 MVP（录制参数输入、开始/停止、导出 GIF、状态展示）
- 状态驱动：`Application` + `Message` + `update` + `view` + `subscription`

## 3) 异步与任务模型

- 异步运行时：`tokio`
- 模式：UI 线程发消息，耗时任务通过 `Command::perform` 异步执行
- 抓帧与编码：使用 `tokio::task::spawn_blocking` 避免阻塞 UI
- 定时采样：`iced::time::every` 按 FPS 触发抓帧

## 4) 录屏采集方案（macOS）

- 当前实现：调用系统命令 `screencapture` 进行区域截图轮询采样
- 采样参数：`x/y/width/height` + `fps`
- 错误处理：对 macOS 屏幕录制权限失败给出明确提示
- 平台策略：当前版本优先支持 macOS

> 说明：当前是“截图轮询”方案，优点是实现快、稳定；后续可升级到 `ScreenCaptureKit` 流式采集以提升性能。

## 5) 图像与 GIF 编码

- 图像解码/处理：`image`
- GIF 编码：`gif`
- 编码策略：
  - 检查帧为空
  - 统一帧尺寸校验
  - 设置循环播放（`Repeat::Infinite`）
  - 由 FPS 换算 GIF delay

## 6) 文件与交互

- 导出路径选择：`rfd`（保存文件对话框）
- 输出格式：`.gif`
- 录制控制增强：
  - 自动停止秒数（`max_seconds`）
  - 手动停止
  - 清空帧缓存

## 7) 打包与分发（macOS）

- 应用打包元数据：`Cargo.toml` 中 `package.metadata.bundle`
- 生成 `.app`：`cargo-bundle`
- 生成 `.dmg`：项目脚本 `scripts/package_dmg.sh`
  - 优先使用 `create-dmg`
  - 无 `create-dmg` 时回退 `hdiutil`

## 8) 当前目录中的关键文件

- 应用入口：`src/main.rs`
- UI 与状态机：`src/app.rs`
- 抓帧实现：`src/capture.rs`
- GIF 编码：`src/encoder.rs`
- 打包脚本：`scripts/package_dmg.sh`
- 使用说明：`readme.md`

## 9) 已验证结果

- `cargo run` 可正常启动并使用
- 打包脚本可生成 `dmg`
- 安装后应用可正常运行

## 10) 后续可选升级

- 将采集层从 `screencapture` 升级为 `ScreenCaptureKit`（更高性能、低延迟）
- 增加区域框选蒙层，减少手填坐标
- 引入更多导出参数（缩放、颜色量化策略、质量档位）
