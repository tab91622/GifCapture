# 区域框选蒙层方案（MVP 级）

目标：减少手输 `x/y/width/height`，提升录制区域选择效率。

## 交互方案

1. 点击“框选区域”进入选择模式（主窗口最小化或隐藏）。
2. 显示全屏半透明蒙层。
3. 鼠标按下记录起点，拖拽实时显示矩形边框与尺寸。
4. 鼠标抬起确认区域，按 `Esc` 取消。
5. 区域回填到 `app` 的 `region_x/y/width/height` 并恢复主窗口。

## 结构建议

- 新增 `src/overlay.rs`：
  - 提供 `select_region() -> Result<CaptureRegion, String>`。
  - 对外只暴露结果，不泄漏平台细节。
- `src/app.rs`：
  - 新增 `Message::PickRegion`、`Message::RegionPicked(...)`。
  - 在非录制/非导出时允许触发框选。

## 实现路线

- 第一阶段（快速落地）：
  - macOS 先用系统命令方案（例如 AppleScript + 截图工具链）实现可用闭环。
- 第二阶段（体验升级）：
  - 使用原生 API 实现实时蒙层与拖拽反馈，降低闪烁与延迟。

## 风险点

- 需要处理多显示器坐标系与 Retina 缩放。
- 需要避免主窗口被误录进结果。
- 需要在权限不足时给出明确引导（屏幕录制/辅助功能）。
